Reassignment Three options exist for reassigning the assignment to a different recipient.
  • reassign = "1"– Creates a reassignment that does not preserve estimate dates. You normally use this option when reassigning to a different recipient. Any reassigned estimates are treated as pending estimates (not completed).
  • reassignPreserve = "1"– Creates a reassignment that preserves estimate dates. You normally use this option when reassigning to a different branch. Any reassigned estimates are treated as completed estimates (and could be returned to carriers).
  • reassignReadOnly = "1"– Sends a read-only copy of the assignment to the recipient.
  • reassignByCarrier = "1"– reassigns an estimate, leaving it active for the sender, but inactive for the carrier
The carrier should also provide the transactionId – which is usually stored on the confirmation log file.
